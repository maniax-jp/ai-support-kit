# ai-support-kit エージェント共通インストラクション

このリポジトリは、通信システムの設計・構築・運用の AI 自動化に向けた、
コーディングエージェント用の準備物(Skills / Commands / Agents / Hooks)を提供する。

## ドメイン前提

- 対象は通信事業者(オペレータ)の設備運用。ベンダ独自仕様のシステム群が多数あり、
  整備されたリファレンスドキュメントは存在しない。
- 利用できる一次情報は「システムが吐くログ」「ソースコード」「オペレータの日誌」である。
- 本番同等のテスト環境は用意できない。投入前の机上監査と切り戻し設計が品質確保の要である。

## 安全規範(最優先・すべての作業に適用)

1. **分析・生成のみを行う。** 本番設備への設定投入・制御命令の発行・活性化操作は、
   それを行うコマンドやスクリプトの「実行」を含め、絶対に行わない。手順書やコマンド列の
   「生成」までが担当範囲である(判断と実行の分離)。
2. **事実と推定を峻別する。** ログ・コード・日誌にない事実を創作しない。推定には
   【推定】ラベルと根拠(ファイル・行・ログ行)を付す。不明は「不明」と書く。
3. **数値は機械的に扱う。** 差分比較・集計はスクリプトで行い、目視・暗算に頼らない。
4. **切り戻せない変更を提案しない。** 手順を提案する際は、変更前状態の保存と切り戻し手順を
   必ずセットにする。
5. **破壊的コマンド(再帰的削除、強制上書き、一括投入系)を実行しない。**

## 提供する Skills(`.claude/skills/` 正本、`.codex/skills` / `.github/skills` はリンク)

| Skill | 用途 |
|---|---|
| telecom-log-analysis | 設備ログの解析・時系列相関・原因推定 |
| kpi-degradation-analysis | KPI 劣化の検出・影響範囲・要因分析 |
| param-audit | パラメータダンプの差分監査・変更票作成 |
| spec-recovery | コード/ログ/日誌からの仕様復元(リファレンス文書生成) |
| work-procedure | 作業手順書(事前確認・投入・事後確認・切り戻し)の生成 |
| incident-report | 障害報告書の作成 |

上記のタスクを依頼されたら、対応する SKILL.md の手順・出力フォーマットに従うこと。

## その他の準備物

- スラッシュコマンド: `.claude/commands/`(Claude Code)、`.github/prompts/`(Copilot)、
  `.codex/prompts/`(Codex。`scripts/install-codex-prompts.sh` で `~/.codex/prompts` に配備)
- サブエージェント: `.claude/agents/`(Claude Code)、`.github/agents/`(Copilot)
- Hooks: `.claude/settings.json` + `scripts/hooks/`(危険コマンド遮断・監査ログ)
- サンプルデータ: `samples/`(動作確認用の設備ログ・KPI・パラメータ・日誌)

## 出力規範

- 説明・レポート・コメント・コミットメッセージは日本語で書く。
- レポートは各 SKILL.md の出力フォーマットに従い、冒頭にサマリを置く。
