# KPI 劣化分析

## 目的

KPI 時系列データから劣化を検出し、「いつから」「どこで」「どの指標が」「どの程度」劣化したかを
定量的に示し、要因候補を導く。フィールドの値を継続収集し改善し続ける運用の起点となる分析。

## 手順

1. **データ構造の把握**: ヘッダ行から指標名・集計単位(セル/装置/エリア)・粒度(日次/時間)を
   把握する。指標の意味が名前から自明でない場合は推定であることを明示する。
2. **ベースラインの確立**: 劣化疑い期間より前の安定期間から各指標の平常値(平均・ばらつき)を
   算出する。
3. **劣化検出**: ベースラインからの乖離が大きい「対象 × 指標 × 期間」を特定する。
   絶対値の閾値をハードコードせず、平常時からの相対変化で判断する。
4. **影響範囲の分析**:
   - 劣化が単一セル/装置か、複数に及ぶか(共通設備・共通作業の示唆)
   - 劣化対象の周辺(隣接セル等)への波及(トラフィック転嫁、負荷上昇)
5. **要因候補の列挙**: 劣化開始時刻と一致するイベント(作業、パラメータ変更、アラーム、
   トラフィック変動、外部要因)を、他情報源が使える場合は突き合わせて列挙する。
6. **レポート作成**: 下記フォーマットで出力する。

## 出力フォーマット

```markdown
# KPI 劣化分析レポート
## 対象データ: <ファイル> (期間 / 集計単位)
## サマリ(劣化の有無・最重要所見を3行以内)
## 劣化検出結果(表: 対象 / 指標 / 平常値 / 劣化後 / 変化量 / 劣化開始日時)
## 影響範囲(波及・ユーザ影響の見立て)
## 要因候補(候補 / 根拠 / 確度)
## 推奨アクション(切り戻し判断・追加収集すべきデータ)
```

## 注意事項

- 数値の集計は暗算せず、必要ならスクリプト(python3 等)で計算して正確を期す。
- 「劣化なし」も重要な結論。無理に異常を見つけようとしない。
- 単日のスパイクと持続的劣化を区別する。持続的劣化のみを「劣化」として報告し、
  スパイクは参考情報とする。
